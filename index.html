🧠 Ejercicio 1: ¿Qué es la programación?

La programación es el proceso sistemático de diseñar y escribir instrucciones que una máquina puede entender y ejecutar. Sus dos componentes fundamentales: los algoritmos y el código.

💻 Ejercicio 2: Algoritmo vs Código

Algoritmo: Un algoritmo es una secuencia precisa de pasos que se sigue para alcanzar un objetivo específico. Es como una receta detallada donde cada paso está perfectamente definido y ordenado.

Código: El código es la implementación práctica de un algoritmo utilizando un lenguaje de programación específico. Es la traducción del algoritmo a un formato que la computadora pueda entender directamente.

📘 Apuntes: Arrays y Cadenas en Python

Resumen de los conceptos y fundamentos más importantes sobre arrays (listas) y cadenas de caracteres en Python:

  • Array (Lista): estructura que almacena múltiples valores bajo un mismo nombre. En Python se implementa mediante listas [].
  • Características: ordenadas, mutables, pueden contener datos homogéneos o heterogéneos.
  • Declaración: numeros = [10, 20, 30]
  • Acceso por índice: empieza en 0. Ejemplo: numeros[0] devuelve el primer elemento.
  • Recorrido: con bucles (for n in lista o for i in range(len(lista))).
  • Métodos comunes: append(), insert(), remove(), pop(), len().
  • Cadenas de texto (Strings): secuencias de caracteres inmutables. Se definen con comillas simples o dobles.
  • Operaciones básicas: len(), concatenación (+), acceso por índice, métodos .lower(), .upper(), .find(), etc.
  • Arrays y cadenas combinados: listas que contienen textos, como nombres = ["Ana", "Luis", "Marta"].

📘 Arrays (Listas) en Python

  • Definición: colección ordenada de elementos.
  • Tipo: en Python se representan con listas.
  • Características: ordenadas, mutables, permiten distintos tipos.
  • Índices: comienzan en 0 y soportan índices negativos.

🔧 Declaración y Acceso

  • Declaración: lista = [10, 20, 30]
  • Lista vacía: lista = []
  • Acceso: lista[0] (primer elemento)
  • Acceso negativo: lista[-1] (último)
  • Modificar: lista[2] = 99

🔁 Recorrido de Arrays

  • Por índice: for i in range(len(lista))
  • For-each: for elemento in lista

📦 Métodos Importantes

  • append(x): añade al final
  • insert(pos, x): inserta en posición
  • remove(x): elimina un elemento
  • pop(i): extrae un elemento por índice
  • extend(lista2): une listas
  • len(lista): longitud

✂️ Slicing (División)

  • Sintaxis: lista[inicio:fin:paso]
  • Desde inicio: lista[:3]
  • Hasta final: lista[2:]
  • Invertir: lista[::-1]

➕ Añadir / Insertar / Unir

  • append(): añadir elemento al final
  • insert(): insertar en una posición
  • extend(): unir dos listas

➖ Eliminar / Extraer

  • remove(): elimina por valor
  • pop(): elimina y devuelve elemento
  • pop(i): elimina por índice

🔍 Búsqueda en Arrays

  • Búsqueda secuencial: recorre la lista elemento a elemento.
  • Búsqueda binaria: requiere lista ordenada.
  • Python: usar in o lista.index().

📊 Ordenación

  • lista.sort(): modifica la lista original
  • sorted(lista): devuelve una copia ordenada
  • Orden sin mayúsculas: sorted(lista, key=str.lower)
  • Por longitud: lista.sort(key=len)

🔎 Comparación de Arrays

  • Dos arrays son iguales si tienen:
    • Misma longitud
    • Mismos valores
    • En el mismo orden
  • Python: lista1 == lista2

📎 Copias de Arrays

  • Incorrecto: copia = original
  • Correcto:
    • copia = original.copy()
    • copia = original[:]
    • Copia manual recorriendo elementos

🧱 Arrays Bidimensionales (Matrices)

  • Representación: listas dentro de listas.
  • Acceso: matriz[fila][columna]
  • Recorrido: bucles anidados

🔤 Cadenas de Caracteres

  • Secuencia inmutable de caracteres.
  • Se crean con 'texto' o "texto".
  • Indexación: cadena[0], cadena[-1]
  • Slicing: cadena[inicio:fin]
  • Invertir: cadena[::-1]

🛠️ Métodos útiles

  • upper(), lower()
  • strip(): eliminar espacios
  • split(): dividir texto
  • replace(a, b)

🧵 Formato de Cadenas

  • f-strings (recomendado):
  • f"Hola {nombre}"

🔢 Tipos de datos y declaración

  • Primitivos: byte, short, int, long, float, double, char, boolean.
  • Declarar e inicializar: double precio = 2.50;. Siempre inicializar antes de usar.
  • Constantes: final double PI = 3.141592;.

🔁 Conversión de tipos

  • Implícita: de tipo menor a mayor (sin pérdida aparente). Ej: int → double.
  • Explícita (casting): int n = (int) 2.9; (pérdida de información posible).

⌨ Entrada y salida

  • Salida: System.out.println("texto").
  • Entrada (Scanner):
    Scanner sc = new Scanner(System.in); int n = sc.nextInt();

➕ Operadores clave

  • Aritméticos: + - * / %
  • Relacionales: == != > < >= <=
  • Lógicos: && || ! (evaluación perezosa)
  • Ternario: cond ? valor1 : valor2

🔀 Control de flujo

  • Condicionales: if, if-else, if-else if, switch.
  • Bucle for: contador conocido. for (int i=0;i<n;i++).
  • Bucle while: condición.
  • do-while: ejecuta al menos una vez.
  • Bucles anidados para matrices/tareas repetidas.

🧩 Funciones / Métodos

  • Método = bloque reutilizable. Sintaxis:
    static tipo nombre(tipo param){ ... }
  • Parámetros por valor: los cambios dentro del método no alteran la variable externa.
  • Devolver valor con return; también existen procedimientos (void).
  • Sobrecarga: mismo nombre, distinta firma (nº/tipo de parámetros).

📝 Ejercicios esenciales (práctica mínima)

  • Pide un número y muestra si es par, positivo y su cuadrado (repetir hasta 0).
  • Calcula la media de tres notas y muestra parte entera y decimal.
  • Implementa una función que devuelva el máximo de dos enteros y otra que calcule factorial con for.
  • Dibuja un triángulo de asteriscos para un n dado (bucles anidados).

✅ Resumen rápido

  • Java = compilar + JVM. Entender este flujo es fundamental.
  • Conocer tipos, declaración, casting y convenciones de nombres.
  • Control de flujo (condicionales/bucles) y métodos son herramientas esenciales antes de POO.

🏗️ ¿Qué es una Clase?

Una clase es una plantilla que permite modelar objetos del mundo real. Define sus datos (atributos) y su comportamiento (métodos).

Ejemplo:


class Persona {
  String nombre;
  int edad;
}

📦 Atributos

  • Representan la información o estado del objeto.
  • Suelen ser sustantivos y usar lowerCamelCase.
  • Se recomienda visibilidad private.

Ejemplo:


private String nombre;
private int edad;
private double estatura;

⚙️ Métodos

Los métodos son funciones definidas dentro de una clase que describen el comportamiento del objeto.

  • Suelen llevar un verbo en el nombre.
  • Pueden modificar atributos del objeto.

Ejemplo:


void cumplirAnios() {
  edad++;
}

🧩 Objetos

Un objeto es una instancia de una clase. Cada objeto tiene sus propios valores para los atributos.

Ejemplo de creación de objetos:


Persona p1 = new Persona();
Persona p2 = new Persona();

🔗 Operador punto (.)

Se utiliza para acceder a los atributos y métodos de un objeto. 


p1.nombre = "Ana";
p1.cumplirAnios();

🧠 Referencia this

La palabra reservada this hace referencia al propio objeto y se usa para diferenciar atributos de variables locales. 


void setEdad(int edad) {
  this.edad = edad;
}

🏗️ Constructores

Los constructores son métodos especiales que se ejecutan al crear un objeto. Sirven para inicializar sus atributos.

Ejemplo:


Persona(String nombre, int edad) {
  this.nombre = nombre;
  this.edad = edad;
}

🔁 Tipos de Constructores

  • Por defecto: si no se define ninguno.
  • Sin parámetros: inicializa valores estándar.
  • Con parámetros: permite personalizar el objeto.

🚨 Referencias y null

  • Los objetos se manejan mediante referencias.
  • Una referencia puede valer null.
  • Acceder a un objeto null produce NullPointerException.

📘 Enumerados (enum)

Un enumerado define un conjunto limitado de valores posibles. 


enum Sexo {
  HOMBRE, MUJER, NO_BINARIO
}

📝 Resumen del Tema

  • La POO permite modelar el mundo real mediante clases y objetos.
  • Las clases definen atributos y métodos.
  • Los objetos son instancias creadas con new.
  • Los constructores inicializan los objetos.
  • El uso correcto de referencias es clave en Java.
Tema 5: Principios de la POO (Encapsulación, Herencia y Polimorfismo)

🚀 Introducción a la UD5

En este tema profundizamos en los pilares que hacen de la POO un paradigma potente para el desarrollo de software a gran escala. Nos enfocamos en cómo proteger los datos y reutilizar el código de forma eficiente.

Ver ejercicios Unidad 5 en GitHub

🛡️ 1. Encapsulación y Visibilidad

Consiste en ocultar el estado interno de un objeto y obligar a que toda interacción se realice a través de métodos públicos. Esto protege la integridad de los datos.

  • private: Solo accesible dentro de la propia clase.
  • protected: Accesible en la propia clase, el mismo paquete y subclases.
  • public: Accesible desde cualquier lugar.
  • Getters y Setters: Métodos estándar para leer y modificar atributos privados con validación.

🌳 2. Herencia (Extends)

Permite crear nuevas clases basadas en clases existentes, heredando sus atributos y métodos. Facilita enormemente la reutilización de código.

  • Superclase: La clase de la que se hereda (ej. Animal).
  • Subclase: La clase que hereda (ej. Perro).
  • Uso de super() para llamar al constructor de la madre.

🎭 3. Polimorfismo y Sobrescritura

Es la capacidad de un objeto de tomar múltiples formas. Una referencia de tipo "Padre" puede apuntar a un objeto "Hijo".

  • Sobrescritura (@Override): Redefinir un método de la madre en la hija para que haga algo específico.
  • Ligadura Dinámica: Java decide en tiempo de ejecución qué método ejecutar según el tipo real del objeto, no de la referencia.

🏗️ 4. Clases y Métodos Abstractos

Se utilizan cuando queremos definir un "concepto" pero no una "implementación" concreta.

  • Clase Abstracta: No se puede instanciar (usar new).
  • Método Abstracto: No tiene cuerpo; obliga a las clases hijas a implementarlo.

🔒 5. Modificador Final

  • Variable final: Se convierte en una constante.
  • Método final: No puede ser sobrescrito por las hijas.
  • Clase final: No puede tener descendencia (no se puede heredar de ella).

Reflexión: ¿Por qué es importante el polimorfismo en sistemas grandes?

Permite que el sistema sea extensible. Podemos añadir nuevas funcionalidades (nuevos tipos de objetos) sin tener que modificar el código que ya funciona, cumpliendo con el principio de abierto/cerrado.
