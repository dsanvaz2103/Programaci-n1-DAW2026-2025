📘 Apuntes: Arrays y Cadenas en Python

Resumen de los conceptos y fundamentos más importantes sobre arrays (listas) y cadenas de caracteres en Python:

  • Array (Lista): estructura que almacena múltiples valores bajo un mismo nombre. En Python se implementa mediante listas [].
  • Características: ordenadas, mutables, pueden contener datos homogéneos o heterogéneos.
  • Declaración: numeros = [10, 20, 30]
  • Acceso por índice: empieza en 0. Ejemplo: numeros[0] devuelve el primer elemento.
  • Recorrido: con bucles (for n in lista o for i in range(len(lista))).
  • Métodos comunes: append(), insert(), remove(), pop(), len().
  • Cadenas de texto (Strings): secuencias de caracteres inmutables. Se definen con comillas simples o dobles.
  • Operaciones básicas: len(), concatenación (+), acceso por índice, métodos .lower(), .upper(), .find(), etc.
  • Arrays y cadenas combinados: listas que contienen textos, como nombres = ["Ana", "Luis", "Marta"].

📘 Arrays (Listas) en Python

  • Definición: colección ordenada de elementos.
  • Tipo: en Python se representan con listas.
  • Características: ordenadas, mutables, permiten distintos tipos.
  • Índices: comienzan en 0 y soportan índices negativos.

🔧 Declaración y Acceso

  • Declaración: lista = [10, 20, 30]
  • Lista vacía: lista = []
  • Acceso: lista[0] (primer elemento)
  • Acceso negativo: lista[-1] (último)
  • Modificar: lista[2] = 99

🔁 Recorrido de Arrays

  • Por índice: for i in range(len(lista))
  • For-each: for elemento in lista

📦 Métodos Importantes

  • append(x): añade al final
  • insert(pos, x): inserta en posición
  • remove(x): elimina un elemento
  • pop(i): extrae un elemento por índice
  • extend(lista2): une listas
  • len(lista): longitud

✂️ Slicing (División)

  • Sintaxis: lista[inicio:fin:paso]
  • Desde inicio: lista[:3]
  • Hasta final: lista[2:]
  • Invertir: lista[::-1]

➕ Añadir / Insertar / Unir

  • append(): añadir elemento al final
  • insert(): insertar en una posición
  • extend(): unir dos listas

➖ Eliminar / Extraer

  • remove(): elimina por valor
  • pop(): elimina y devuelve elemento
  • pop(i): elimina por índice

🔍 Búsqueda en Arrays

  • Búsqueda secuencial: recorre la lista elemento a elemento.
  • Búsqueda binaria: requiere lista ordenada.
  • Python: usar in o lista.index().

📊 Ordenación

  • lista.sort(): modifica la lista original
  • sorted(lista): devuelve una copia ordenada
  • Orden sin mayúsculas: sorted(lista, key=str.lower)
  • Por longitud: lista.sort(key=len)

🔎 Comparación de Arrays

  • Dos arrays son iguales si tienen:
    • Misma longitud
    • Mismos valores
    • En el mismo orden
  • Python: lista1 == lista2

📎 Copias de Arrays

  • Incorrecto: copia = original
  • Correcto:
    • copia = original.copy()
    • copia = original[:]
    • Copia manual recorriendo elementos

🧱 Arrays Bidimensionales (Matrices)

  • Representación: listas dentro de listas.
  • Acceso: matriz[fila][columna]
  • Recorrido: bucles anidados

🔤 Cadenas de Caracteres

  • Secuencia inmutable de caracteres.
  • Se crean con 'texto' o "texto".
  • Indexación: cadena[0], cadena[-1]
  • Slicing: cadena[inicio:fin]
  • Invertir: cadena[::-1]

🛠️ Métodos útiles

  • upper(), lower()
  • strip(): eliminar espacios
  • split(): dividir texto
  • replace(a, b)

🧵 Formato de Cadenas

  • f-strings (recomendado):
  • f"Hola {nombre}"

🔢 Tipos de datos y declaración

  • Primitivos: byte, short, int, long, float, double, char, boolean.
  • Declarar e inicializar: double precio = 2.50;. Siempre inicializar antes de usar.
  • Constantes: final double PI = 3.141592;.

🔁 Conversión de tipos

  • Implícita: de tipo menor a mayor (sin pérdida aparente). Ej: int → double.
  • Explícita (casting): int n = (int) 2.9; (pérdida de información posible).

⌨ Entrada y salida

  • Salida: System.out.println("texto").
  • Entrada (Scanner):
    Scanner sc = new Scanner(System.in); int n = sc.nextInt();

➕ Operadores clave

  • Aritméticos: + - * / %
  • Relacionales: == != > < >= <=
  • Lógicos: && || ! (evaluación perezosa)
  • Ternario: cond ? valor1 : valor2

🔀 Control de flujo

  • Condicionales: if, if-else, if-else if, switch.
  • Bucle for: contador conocido. for (int i=0;i<n;i++).
  • Bucle while: condición.
  • do-while: ejecuta al menos una vez.
  • Bucles anidados para matrices/tareas repetidas.

🧩 Funciones / Métodos

  • Método = bloque reutilizable. Sintaxis:
    static tipo nombre(tipo param){ ... }
  • Parámetros por valor: los cambios dentro del método no alteran la variable externa.
  • Devolver valor con return; también existen procedimientos (void).
  • Sobrecarga: mismo nombre, distinta firma (nº/tipo de parámetros).

📝 Ejercicios esenciales (práctica mínima)

  • Pide un número y muestra si es par, positivo y su cuadrado (repetir hasta 0).
  • Calcula la media de tres notas y muestra parte entera y decimal.
  • Implementa una función que devuelva el máximo de dos enteros y otra que calcule factorial con for.
  • Dibuja un triángulo de asteriscos para un n dado (bucles anidados).

✅ Resumen rápido

  • Java = compilar + JVM. Entender este flujo es fundamental.
  • Conocer tipos, declaración, casting y convenciones de nombres.
  • Control de flujo (condicionales/bucles) y métodos son herramientas esenciales antes de POO.
Tema 4: Programación Orientada a Objetos — Clases y Objetos

🎯 Objetivos del Tema

  • Comprender los conceptos de clase y objeto.
  • Crear clases identificando correctamente atributos y métodos.
  • Instanciar y utilizar objetos en programas Java.
  • Utilizar constructores y referencias a objetos.

💻 Ejercicios desarrollados en clase

Los ejercicios realizados y corregidos durante las clases se encuentran disponibles en el repositorio de GitHub:

Ver ejercicios Unidad 4 en GitHub

🏗️ ¿Qué es una Clase?

Una clase es una plantilla que permite modelar objetos del mundo real. Define sus datos (atributos) y su comportamiento (métodos).

Ejemplo:


class Persona {
  String nombre;
  int edad;
}

📦 Atributos

  • Representan la información o estado del objeto.
  • Suelen ser sustantivos y usar lowerCamelCase.
  • Se recomienda visibilidad private.

Ejemplo:


private String nombre;
private int edad;
private double estatura;

⚙️ Métodos

Los métodos son funciones definidas dentro de una clase que describen el comportamiento del objeto.

  • Suelen llevar un verbo en el nombre.
  • Pueden modificar atributos del objeto.

Ejemplo:


void cumplirAnios() {
  edad++;
}

🧩 Objetos

Un objeto es una instancia de una clase. Cada objeto tiene sus propios valores para los atributos.

Ejemplo de creación de objetos:


Persona p1 = new Persona();
Persona p2 = new Persona();

🔗 Operador punto (.)

Se utiliza para acceder a los atributos y métodos de un objeto. 


p1.nombre = "Ana";
p1.cumplirAnios();

🧠 Referencia this

La palabra reservada this hace referencia al propio objeto y se usa para diferenciar atributos de variables locales. 


void setEdad(int edad) {
  this.edad = edad;
}

🏗️ Constructores

Los constructores son métodos especiales que se ejecutan al crear un objeto. Sirven para inicializar sus atributos.

Ejemplo:


Persona(String nombre, int edad) {
  this.nombre = nombre;
  this.edad = edad;
}

🔁 Tipos de Constructores

  • Por defecto: si no se define ninguno.
  • Sin parámetros: inicializa valores estándar.
  • Con parámetros: permite personalizar el objeto.

🚨 Referencias y null

  • Los objetos se manejan mediante referencias.
  • Una referencia puede valer null.
  • Acceder a un objeto null produce NullPointerException.

📘 Enumerados (enum)

Un enumerado define un conjunto limitado de valores posibles. 


enum Sexo {
  HOMBRE, MUJER, NO_BINARIO
}

📝 Resumen del Tema

  • La POO permite modelar el mundo real mediante clases y objetos.
  • Las clases definen atributos y métodos.
  • Los objetos son instancias creadas con new.
  • Los constructores inicializan los objetos.
  • El uso correcto de referencias es clave en Java.
Tema 5: Principios de la POO (Encapsulación, Herencia y Polimorfismo)

🚀 Introducción a la UD5

En este tema profundizamos en los pilares que hacen de la POO un paradigma potente para el desarrollo de software a gran escala. Nos enfocamos en cómo proteger los datos y reutilizar el código de forma eficiente.

Ver ejercicios Unidad 5 en GitHub

🛡️ 1. Encapsulación y Visibilidad

Consiste en ocultar el estado interno de un objeto y obligar a que toda interacción se realice a través de métodos públicos. Esto protege la integridad de los datos.

  • private: Solo accesible dentro de la propia clase.
  • protected: Accesible en la propia clase, el mismo paquete y subclases.
  • public: Accesible desde cualquier lugar.
  • Getters y Setters: Métodos estándar para leer y modificar atributos privados con validación.

🌳 2. Herencia (Extends)

Permite crear nuevas clases basadas en clases existentes, heredando sus atributos y métodos. Facilita enormemente la reutilización de código.

  • Superclase: La clase de la que se hereda (ej. Animal).
  • Subclase: La clase que hereda (ej. Perro).
  • Uso de super() para llamar al constructor de la madre.

🎭 3. Polimorfismo y Sobrescritura

Es la capacidad de un objeto de tomar múltiples formas. Una referencia de tipo "Padre" puede apuntar a un objeto "Hijo".

  • Sobrescritura (@Override): Redefinir un método de la madre en la hija para que haga algo específico.
  • Ligadura Dinámica: Java decide en tiempo de ejecución qué método ejecutar según el tipo real del objeto, no de la referencia.

🏗️ 4. Clases y Métodos Abstractos

Se utilizan cuando queremos definir un "concepto" pero no una "implementación" concreta.

  • Clase Abstracta: No se puede instanciar (usar new).
  • Método Abstracto: No tiene cuerpo; obliga a las clases hijas a implementarlo.

🔒 5. Modificador Final

  • Variable final: Se convierte en una constante.
  • Método final: No puede ser sobrescrito por las hijas.
  • Clase final: No puede tener descendencia (no se puede heredar de ella).

Reflexión: ¿Por qué es importante el polimorfismo en sistemas grandes?

Permite que el sistema sea extensible. Podemos añadir nuevas funcionalidades (nuevos tipos de objetos) sin tener que modificar el código que ya funciona, cumpliendo con el principio de abierto/cerrado.

Tema 6: Interfaces

🔗 Introducción a la UD6

En esta unidad exploramos las interfaces, un mecanismo fundamental en Java para establecer contratos de comportamiento y permitir que clases de diferentes jerarquías compartan capacidades comunes.

Ver ejercicios Unidad 6 en GitHub

📜 1. El Concepto de Interfaz

Una interfaz es un "contrato" que indica qué debe hacer una clase, pero no cómo hacerlo. Es el nivel máximo de abstracción en Java.

  • No tienen estado (no guardan datos variables, solo constantes).
  • No se pueden instanciar (no puedes hacer new de una interfaz).
  • Las clases "implementan" interfaces usando la palabra reservada implements.

⚙️ 2. Componentes de una Interfaz

Por defecto, Java asume ciertos modificadores para ahorrar código y mantener la limpieza:

  • Métodos: Son implícitamente public abstract. Solo definen la firma.
  • Constantes: Son implícitamente public static final.
  • Métodos Default (Java 8+): Permiten añadir lógica por defecto sin romper las clases que ya implementaban la interfaz.
  • Métodos Static: Métodos de utilidad que pertenecen a la propia interfaz.

🧬 3. Herencia Múltiple de Tipo

A diferencia de las clases (donde solo puedes heredar de una), Java permite que una clase implemente infinitas interfaces.


public class RobotDomestico implements InteligenciaArtificial, Electrodomestico {
    // Debe implementar los métodos de AMBAS interfaces
}

🎭 4. Interfaces y Polimorfismo

Es la mejor práctica de diseño: Declarar con la interfaz y crear con la clase. Esto permite cambiar la implementación fácilmente sin romper el resto del programa.


// Recomendado:
Vehiculo miCoche = new Coche();
miCoche.acelerar(20);

// Esto permite que luego puedas hacer:
miCoche = new Moto(); // Sigue funcionando porque ambos son Vehículos

⚖️ 5. Interfaz vs Clase Abstracta

¿Cuándo usar cada una?

  • Clase Abstracta: Cuando hay una relación clara de "es un" (ej. Un Perro es un Animal). Se comparte código y atributos.
  • Interfaz: Cuando hay una relación de "es capaz de" (ej. Un Coche es capaz de de ser Remolcable). Une objetos que no tienen nada que ver entre sí.

Reflexión: ¿Cuál es la mayor ventaja de las interfaces?

La flexibilidad. Permiten desacoplar el código: el programa sabe qué esperar de un objeto (el contrato) sin importarle quién sea ese objeto o cómo funcione internamente.

Tema 7: Control y Manejo de Excepciones

⚠️ Introducción a la UD7

Una excepción es un evento que ocurre durante la ejecución de un programa y que interrumpe su flujo normal de instrucciones. Aprender a manejarlas es vital para crear aplicaciones robustas y "a prueba de fallos".

Ver ejercicios Unidad 7 en GitHub

🏗️ 1. Jerarquía de Excepciones

En Java, todas las excepciones derivan de la clase Throwable. Se dividen principalmente en dos grupos:

  • Error: Problemas graves del sistema (ej. StackOverflowError). No se deben capturar.
  • Exception: Problemas que la aplicación puede manejar.
    • Checked: Obligatorio gestionarlas (ej. IOException).
    • Unchecked (Runtime): Errores de lógica (ej. NullPointerException, ArithmeticException).

🛡️ 2. El bloque Try-Catch-Finally

Es la estructura fundamental para capturar errores y evitar que el programa se cierre inesperadamente.

  • try: Código que puede lanzar una excepción.
  • catch: Código que se ejecuta si ocurre el error.
  • finally: Código que se ejecuta siempre (útil para cerrar archivos o bases de datos).

try {
    int resultado = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("¡No puedes dividir por cero!");
} finally {
    System.out.println("Limpieza de recursos...");
}

📤 3. Lanzamiento y Propagación (Throw/Throws)

A veces no queremos manejar el error en el método actual, sino avisar a quien llamó al método.

  • throw: Lanza una excepción de forma manual.
  • throws: Se usa en la firma del método para indicar que este puede lanzar una excepción.

public void validarEdad(int edad) throws Exception {
    if (edad < 18) {
        throw new Exception("Menor de edad no permitido");
    }
}

✨ 4. Buenas Prácticas

Para escribir código profesional con excepciones debemos seguir estos principios:

  • No captures Exception: Sé lo más específico posible (ej. usa FileNotFoundException en vez de Exception).
  • No dejes el catch vacío: Al menos imprime el error para saber qué ha pasado.
  • Cierra recursos: Usa el bloque finally o try-with-resources.
  • Crea tus propias excepciones: Si la lógica de tu negocio lo requiere (ej. SaldoInsuficienteException).

Reflexión: ¿Por qué es mejor lanzar una excepción que devolver un valor de error (como -1)?

Porque las excepciones no se pueden ignorar fácilmente. Obligan al programador a decidir cómo manejar el error o a propagarlo, mientras que un valor de retorno puede pasar desapercibido y causar errores más graves adelante.

Tema 8: Colecciones (JCF) y Tipos Abstractos de Datos

📚 Introducción a la UD8

El Java Collections Framework (JCF) es un conjunto de interfaces y clases que permiten almacenar y manipular grupos de objetos de forma eficiente. En lugar de usar arrays fijos, usamos estructuras dinámicas que crecen según nuestras necesidades.

Ver ejercicios Unidad 8 en GitHub

🏗️ 1. Jerarquía de la Interfaz Collection

Casi todas las colecciones heredan de Collection. Las tres ramas principales son:

  • List (Listas): Permiten elementos duplicados y mantienen el orden de inserción. (ej. ArrayList, LinkedList).
  • Set (Conjuntos): No permiten duplicados. Ideales para colecciones de elementos únicos. (ej. HashSet, TreeSet).
  • Queue (Colas): Diseñadas para procesar elementos en un orden específico (FIFO).

🗺️ 2. Mapas (Key-Value)

Los mapas no heredan de Collection. Almacenan parejas de clave-valor. Cada clave es única y apunta a un valor.

  • HashMap: Rápido, no garantiza orden.
  • TreeMap: Mantiene las claves ordenadas de forma natural.
  • LinkedHashMap: Mantiene el orden de inserción de las claves.

Map<String, Integer> edades = new HashMap<>();
edades.put("Ana", 25);
edades.put("Luis", 30);
System.out.println(edades.get("Ana")); // Imprime 25

🔄 3. Iteración y Genéricos

Para recorrer colecciones de forma segura usamos el bucle for-each o Iteradores. Los Genéricos <T> aseguran que la colección solo contenga el tipo de dato especificado.


List<String> nombres = new ArrayList<>();
nombres.add("Java");
nombres.add("Python");

for (String nombre : nombres) {
    System.out.println(nombre);
}

🛠️ 4. La clase útil Collections

Java proporciona la clase java.util.Collections con métodos estáticos para operar sobre las colecciones:

  • sort(lista): Ordena los elementos.
  • shuffle(lista): Desordena los elementos aleatoriamente.
  • reverse(lista): Invierte el orden.
  • max(col) / min(col): Encuentra el valor mayor o menor.

⚖️ 5. Comparable vs Comparator

Para ordenar objetos personalizados (como una lista de Alumnos):

  • Comparable: Define el orden natural de la propia clase (método compareTo).
  • Comparator: Define órdenes alternativos de forma externa (ej. ordenar por nombre o por nota).

Reflexión: ¿Cuándo usaría un HashSet en lugar de un ArrayList?

Usaría un HashSet cuando la prioridad sea evitar duplicados y necesite una búsqueda muy rápida, ya que el ArrayList permite repetidos y es más lento para comprobar si un elemento ya existe en listas muy grandes.

🔌 Arquitectura JDBC (Java Database Connectivity)

JDBC es una API estándar de Java que permite la ejecución de operaciones SQL de forma independiente a la base de datos utilizada.

  • Componentes Clave: Se compone del gestor de controladores (DriverManager) y los drivers específicos proporcionados por los fabricantes (MySQL, PostgreSQL, Oracle).
  • Protocolo de Conexión: Se establece mediante una cadena URL específica. Ejemplo: jdbc:mysql://localhost:3306/mi_base_datos.
  • Ciclo de vida: Es imperativo abrir, operar y cerrar los flujos de comunicación utilizando bloques try-with-resources para evitar fugas de memoria (Memory Leaks).

⚡ Gestión de Sentencias: Statements y Seguridad

La ejecución de sentencias SQL en Java requiere la correcta elección de interfaces para optimizar el rendimiento y blindar la aplicación.

  • Statement: Utilizado para consultas estáticas sin parámetros. Presenta serios riesgos de seguridad.
  • PreparedStatement: Compila la estructura de la consulta en el servidor de Base de Datos antes de recibir los valores. Es obligatorio para parametrizar entradas, aumentando el rendimiento y previniendo ataques de Inyección SQL.
  • CallableStatement: Interfaz heredada de las anteriores diseñada específicamente para invocar procedimientos almacenados y funciones dentro de la Base de Datos.

📊 Procesamiento de Datos con ResultSet

Los resultados devueltos por las consultas de selección estructuradas (SELECT) se gestionan mediante cursores virtuales en Java.

  • Estructura de cursor: La interfaz ResultSet mantiene un puntero apuntando inicialmente justo antes de la primera fila de datos.
  • Recorrido Dinámico: El método next() desplaza el cursor a la siguiente fila. Devuelve false cuando no quedan registros, integrándose perfectamente en bucles while (rs.next()) { ... }.
  • Extracción Tipo-Específica: Los datos se recuperan mediante métodos específicos por columna, ya sea por índice numérico o por nombre de etiqueta (ej: rs.getInt("id"), rs.getString("nombre")).

🏗️ El Patrón de Diseño DAO (Data Access Object)

Para mantener los principios de la POO limpios, se aísla por completo la lógica de negocio de los mecanismos físicos de persistencia.

  • Abstracción: Se define una interface con las operaciones CRUD básicas (Create, Read, Update, Delete) de la entidad.
  • Implementación (DAO Impl): Clase concreta que implementa la interfaz utilizando el código JDBC puro. Facilita cambiar de motor de base de datos sin alterar el resto de la aplicación.

📝 Resumen del Tema 9

  • JDBC actúa como puente unificado entre las aplicaciones Java y cualquier motor de base de datos relacional.
  • La seguridad mediante PreparedStatement es innegociable en entornos de producción.
  • El mapeo de datos se realiza fila a fila transformando los registros del ResultSet en colecciones de objetos Java.
  • El patrón DAO garantiza que el código sea modular, escalable y mantenible a largo plazo.
Tema 10: Persistencia y Bases de Datos

🗄️ Introducción a la UD10

En esta unidad abordamos el problema de la persistencia: cómo hacer que la información de nuestras aplicaciones sobreviva entre distintas ejecuciones, evolucionando desde el almacenamiento en archivos planos hacia el uso avanzado de bases de datos orientadas a objetos y relacionales.

Ver ejercicios Unidad 10 en GitHub

💎 1. Bases de Datos Orientadas a Objetos (BBDDOO) y JPA

A diferencia del modelo relacional, las BBDDOO almacenan la información directamente en forma de objetos, eliminando la necesidad de traducir los datos. Java utiliza la especificación JPA (Java Persistence API) para gestionar este mapeo de forma transparente.

  • ObjectDB: SGBD orientado a objetos nativo utilizado para integrar persistencia directa sin SQL.
  • Anotaciones Clave: Uso de @Entity para marcar clases persistibles, @Id para la clave primaria y @Transient para excluir campos de la base de datos.
  • EntityManager: Interfaz fundamental encargada de controlar el ciclo de vida de los objetos y las transacciones (persist, find, remove).

🔍 2. Consultas con JPQL

Java Persistence Query Language (JPQL) permite realizar consultas sobre las entidades y sus atributos en lugar de operar sobre las tablas físicas de la base de datos.

  • Operaciones de búsqueda avanzadas mediante SELECT, UPDATE y DELETE estructurados bajo lógica de objetos.
  • Uso de parámetros nombrados (ej. :variable) para realizar filtros dinámicos seguros dentro del código de persistencia.

TypedQuery<Evento> q = em.createQuery(
    "SELECT e FROM Evento e WHERE e.precio > :p", Evento.class
);
q.setParameter("p", 25.0);

⚡ 3. JDBC Relacional y Ejecución en Lote (Batch)

Para interactuar de forma nativa con bases de datos relacionales como MariaDB, empleamos JDBC. Aprendimos a optimizar los tiempos de respuesta del servidor reduciendo el tráfico de peticiones.

  • Manejo de Recursos: Conexión explícita y segura a través de DriverManager.getConnection().
  • Procesamiento Batch: Agrupación de múltiples sentencias de actualización SQL en un único lote a través de addBatch() y su posterior ejecución masiva con executeBatch().

📊 4. Extracción de Metadatos

La API de JDBC no solo permite manipular registros, sino que también ofrece herramientas avanzadas para auditar la estructura física interna del propio sistema gestor.

  • Uso de la interfaz DatabaseMetaData para extraer dinámicamente información estructural sin lanzar consultas de selección complejas.
  • Permite recuperar de forma programática las tablas de la base de datos, las columnas de una entidad, la URL de conexión y la versión del driver o SGBD utilizado.

Reflexión: ¿Qué diferencia a JPA frente a JDBC a nivel arquitectónico?

JPA nos permite trabajar bajo el paradigma puro de la Orientación a Objetos, abstrayéndonos por completo del lenguaje SQL. JDBC, por el contrario, nos sitúa a bajo nivel otorgándonos un control manual, absoluto y milimétrico sobre las consultas y el rendimiento físico del motor de la base de datos relacional.

Tema 11: Interfaces Gráficas de Usuario (GUI)

🚀 Introducción a la UD11

En este tema abandonamos el flujo secuencial de las aplicaciones en consola (CUI) para dar el salto al desarrollo de interfaces visuales e interactivas de ventanas (GUI), fundamentadas en el paradigma de la programación dirigida por eventos.

Ver ejercicios Unidad 11 en GitHub

🖼️ 1. Librerías Gráficas, Componentes y Contenedores

Estudiamos el ecosistema y evolución de las herramientas nativas de Java (AWT, Swing y JavaFX), asimilando que toda interfaz se edifica mediante una jerarquía estricta de elementos.

  • Componentes (Controles): Elementos visuales básicos de interacción interactiva (JButton, JTextField, JLabel).
  • Contenedores: Componentes especiales cuya función es albergar y agrupar a otros controles en su interior (JFrame como ventana principal y JPanel para subdivisiones).

📐 2. Gestores de Distribución (Layout Managers)

Un Layout Manager es el motor matemático encargado de calcular de forma dinámica el tamaño y la localización de los componentes, garantizando que la interfaz sea fluida y elástica.

  • FlowLayout: Posiciona elementos de forma secuencial, uno al lado del otro, saltando de línea si no hay espacio.
  • BorderLayout: Divide el área del contenedor en 5 regiones estratégicas (North, South, East, West y Center).
  • GridLayout: Organiza el espacio en una cuadrícula rígida de filas y columnas uniformes.
  • Diseño Absoluto (null): Desactiva los gestores automáticos para forzar posiciones estáticas manuales mediante coordenadas setBounds(x, y, ancho, alto).

🛡️ 3. Modelo de Delegación de Eventos

La interacción del usuario no sigue un orden lineal. El sistema reacciona de manera asíncrona mediante el patrón arquitectónico Observer.

  • Event Source: El componente visual que sufre la acción física (ej. un clic en un botón) y genera un objeto de evento (ActionEvent).
  • Event Listener (Oyente): Interfaz dedicada a "escuchar" el evento. Para activarlo, debe registrarse explícitamente en el componente mediante métodos como addActionListener().

// Estructura clásica implementando la interfaz ActionListener
public class MiVentana extends JFrame implements ActionListener {
    public MiVentana() {
        boton.addActionListener(this);
    }
    @Override
    public void actionPerformed(ActionEvent e) {
        // Código de respuesta al evento
    }
}

📋 4. Construcción Jerárquica de Menús

Los sistemas de menús desplegables complejos de escritorio siguen un ensamblaje inverso y anidado de tres niveles:

  • JMenuItem: Las opciones u hojas individuales del menú final sobre las que se puede hacer clic.
  • JMenu: Los bloques desplegables intermedios que agrupan a los items independientes.
  • JMenuBar: La barra superior horizontal contenedora que se ancla directamente al armazón del JFrame.

Reflexión: ¿Por qué es una mala práctica abusar del diseño absoluto (null layout) en interfaces de escritorio?

Porque las aplicaciones con posicionamiento absoluto pierden la adaptabilidad. Si el usuario redimensiona la ventana, maximiza la pantalla o ejecuta el programa en un sistema operativo con una resolución o escala de fuentes distinta, los componentes visuales se descolocarán, se solaparán o quedarán parcialmente invisibles.
